# SEO-Monthly-Babli

Automated monthly **SEO Intelligence Report** for **Babli** (babli.in), generated by GreenHonchos.

- **Reporting period:** 01 May 2026 → 29 May 2026
- **Comparison period:** 01 Apr 2026 → 29 Apr 2026
- **Live report:** `reports/index.html` (published via GitHub Pages)

## Structure
```
reports/index.html      Rendered report (visually identical to the approved master template)
templates/report.html.j2  Master template
data/semrush.json       Parsed SEMrush position tracking
data/raw/*.json         Live source pulls (GA4, GSC, Google Ads, Meta, Bing, Clarity, Shopify)
logs/delivery-log.csv   Email + publish delivery log
```

## Data sources (live)
| Section | Source |
|---|---|
| Organic overview, trend, new/returning, device, city, funnel, content, ecommerce | GA4 `354488080` (Windsor.ai) |
| Search visibility, brand/non-brand, landing pages | Google Search Console `sc-domain:babli.in` (Windsor.ai) |
| SERP & keywords | SEMrush position-tracking export |
| Paid (SEO vs Paid) | Google Ads `852-589-3235` + Meta Ads (Windsor.ai) |
| Commerce cross-check | Shopify `986f2c.myshopify.com` (Windsor.ai) |
| Bing card | Bing Webmaster API |
| Clarity card | Microsoft Clarity Data Export API |
| Core Web Vitals | CrUX values (per approved template) |

## Data-pending cells (not fabricated)
- SEMrush **footprint (985 kw)** & **featured snippets** — needs Domain-Organic export
- GSC **index coverage** — Windsor connector exposes Search Analytics only
- Google Ads **brand-keyword overlap** — connector exposes match-type, not keyword text
- Shopify **stockouts** — inventory not exposed by the connected feed
- **Cannibalisation** — needs multi-URL-per-query export

## Regenerate
```
python3 _build/render.py SEO-Monthly-Babli Babli "<logo_url>" babli.in
```
